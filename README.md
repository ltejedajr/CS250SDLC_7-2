CS 250 Final Project Reflection

How do I interpret user needs and implement them into a program? How does creating user stories help with this?

Interpreting user needs begins with understanding the problem the user is trying to solve. This means gathering requirements, asking clarifying questions, and identifying the features that provide the most value. In software development, user needs must be translated into technical requirements that can be built, tested, and improved.

User stories help simplify this process by framing requirements from the user’s perspective. For example, instead of focusing immediately on code, a story like “As a traveler, I want to search destinations so I can compare vacation options” makes the goal clear. This approach helps break large requirements into smaller, manageable tasks while keeping the user at the center of development.

How do I approach developing programs? What Agile processes do I hope to incorporate into my future development work?

I approach developing programs by first understanding the structure of the problem, then breaking it into smaller parts that can be developed incrementally. I focus on creating a strong foundation before adding complexity, which helps reduce errors and makes debugging easier.

Agile has shown me the importance of flexibility and continuous improvement. In future projects, I plan to incorporate sprint planning, daily standups, backlog refinement, and retrospectives. These processes improve organization, communication, and adaptability. I also value iterative delivery because it allows me to build working software faster and gather feedback early.

What does it mean to be a good team member in software development?

Being a good team member means communicating clearly, collaborating effectively, and taking responsibility for assigned work. Strong team members support each other by sharing knowledge, helping solve problems, and staying accountable to deadlines.

It also means being adaptable when requirements change and staying open to feedback. In Agile environments, teamwork is critical because each person contributes to the success of the sprint. A good team member not only focuses on writing code but also on helping the team achieve its overall goals.
